## 二、P0 级（收录阻断 · 已修复 + 用户动作）

### P0-1 预渲染脚本 TDZ 错误致全部页面空壳 → GEO 收录归零 ✅已修复

- **现象**（本地实跑 `node scripts/prerender.js --no-server`）：
  ```
  预渲染失败: ReferenceError: Cannot access 'ROUTES' before initialization
      at main (file:///.../prerender.js:154:29)
  ```
- **根因**：`prerender.js` 第 154 行 `console.log(... ROUTES.length ...)` 在第 158 行 `const ROUTES = await buildRoutes()` 声明**之前**引用了 `ROUTES`，触发 ES 模块临时死区（TDZ）。预渲染全程失败 → `dist/` 仅剩 `vite build` 生成的纯 SPA 空壳 → AI 爬虫（不执行 JS）拿到 `<div id="app"></div>` → **GEO 一切归零**。
- **这是前序报告未发现的 GEO 致命项**：前序报告称"SSG 预渲染 11/11 成功"，但本次实测在引入动态路由后暴露 TDZ；即便未引入动态路由，任何对 `ROUTES` 的前置引用都会触发同样错误。
- **影响**：文心/豆包/通义/ChatGPT/Perplexity 抓取所有页面均为空壳，结构化数据、FAQ、正文全部不可见。
- **修复**：将 `const ROUTES = await buildRoutes()` 调用上移至浏览器启动后、循环之前，消除 TDZ。
- **验证**（修复后实跑）：
  ```
  成功率: 25/25  |  HTML 总量: ~243KB
  ✓ /                    首页           9.5KB
  ✓ /about               关于我们         12.5KB
  ✓ /faq                 FAQ          18.3KB
  ✓ /news/1..20          资讯详情        8.6KB
  ```

### P0-2 预渲染路由硬编码 /news/1..6，14 篇详情页空壳 ✅已修复

- **现象**：`prerender.js` 的 `ROUTES` 硬编码 `/news/1`~`/news/6`，但资讯已扩至 20 篇 → 第 7~20 篇详情页**无预渲染快照**，AI/搜索引擎访问这些 URL 拿到 SPA 空壳。
- **修复**：新增 `fetchArticleRoutes()`，从 `/api/articles/?page=N&page_size=100` 动态拉取全部已发布文章（含 slug 回退 ID），覆盖全部详情页。
- **验证**：预渲染日志 `静态路由 5 条 + 资讯详情 20 条 ✓ 共 25 条路由`，25/25 成功。

### P0-3 站长验证 / 统计 ID 仍为占位符（延续项） ⏳用户动作

- **现象**（`GET /api/site-config/info/`）：`baidu_verify`/`qihu_verify`/`sogou_verify`/`google_verify`/`baidu_tongji`/`gtm_id`/`google_ga_id` 全为占位串。
- **影响**：百度/360/搜狗/Google 站长验证失败 → 无法提交 sitemap、无法开启主动推送 → **搜索引擎不收录或延迟收录**。
- **状态**：代码链路已通（`seo.js` 正确注入 meta 与统计脚本），**仅缺真实凭证**。这是整个收录优化的第一阻断点。
- **用户动作**：备案通过后，在 Django Admin（`/admin`，账号 `admin`/`Laborsaving2024!`，首次登录后请立即改密）「站点配置」填入 7 项真实值。

### P1-2 Schema 结构化数据不完整（GEO §3.2） ✅已修复

- **缺失**：`Organization` Schema 的 `areaServed` 为文本而非 `Country CN`；缺 `sameAs`（EEAT 权威外链底座）；缺 `Service`+`Offer` 招商语义。
- **修复**（后端 `seo_urls.py` + 前端 `useJsonLd.js` 对齐）：
  - `areaServed` → `{"@type":"Country","name":"CN"}`（全国，与 `address` 真实辽宁地址分离）
  - 新增 `makesOffer`：`{"@type":"Offer","category":"区域代理授权","description":"区域保护、价格政策、产品培训、售后技术支持"}`
  - 新增 `sameAs` 字段（`SiteConfig.same_as`，逗号/换行分隔，空时不输出，避免空数组）
- **验证**：`curl /schema/organization.json` 确认三类结构化数据齐备。

### P1-3 转化表单缺招商字段与线索归因（SEO §9.1 + GEO §3.4） ✅已修复

- **缺失**：`Contact` 模型 + `Contact.vue` 仅有 姓名/电话/邮箱/留言；缺 **意向代理区域、投资预算区间、lead_source**。
- **修复**：
  - `Contact` 模型新增 `intent_region`/`budget_range`/`lead_source`（迁移 `0008`）
  - `ContactSerializer` 暴露新字段
  - `Contact.vue` 增加两个下拉选择（代理区域/预算区间）+ `detectLeadSource()` 自动归因（ai/search/direct/other）
  - 事件埋点对齐招商语义：百度统计 `_trackEvent('招商线索','提交','代理意向')`，GTM `dataLayer.push({event:'lead_submit',leadType:'franchise',leadSource:...})`
- **验证**：POST 表单成功入库，`lead_source=ai, intent_region=华东, budget_range=30-50万` 均已存储。

### P1-4 缺 canonical / OpenGraph ✅已修复

- **缺失**：`index.html` 与 `seo.js` 未注入 `<link rel="canonical">` 与 OG 标签（深度报告 §2.2）。
- **修复**：`seo.js` 新增 `setCanonicalOg()`，按路由动态注入 canonical + OG（type/url/title/description/og:image/site_name），router 守卫传入 `path`。
- **效果**：避免重复收录，提升社交分享预览质量。

### P1-5 robots.txt 缺 Bytespider + 双源不一致（GEO §3.3） ✅已修复

- **缺失**：静态 `public/robots.txt` 与后端 `seo_urls.robots_txt` 均缺 `Bytespider`（字节/AI 引擎），后端版还缺 `Claude-Web`、`360Spider`，且 sitemap 用相对路径。
- **修复**：统一双源，补齐 `GPTBot/ChatGPT-User/CCBot/Claude-Web/PerplexityBot/Bytespider`，新增 `Allow: /llms.txt` + `Allow: /llms-full.txt`，sitemap 用绝对路径 `https://laborsaving-arm.cn/sitemap.xml`。
- **验证**：`curl /robots.txt` 含 `Bytespider`、`360Spider`、`Allow: /llms.txt`。

### P1-6 `/assets/*` 被 spa_fallback 误吞为 HTML ✅已修复

- **现象**：`config/urls.py` 的 `spa_fallback` 正则 `^(?!api/|admin/|static/|media/).*` 未排除 `/assets/`，导致生产环境下 `/assets/*.js` 被兜底返回 `dist/index.html`（HTML），前端 JS 加载失败、页面交互失效。
- **修复**：正则改为 `^(?!api/|admin/|static/|media/|assets/|favicon\.ico).*$`。
- **验证**：`/assets/*.js` 不再返回 HTML（返回 Django staticfiles 服务或 404，由 STATICFILES_DIRS + Nginx 映射服务）。

### P3-1 FAQ 招商化与合规（内容运营）

- **现状**：FAQ 15 条，结构符合"问题→答案→详情"，全站**无"稳赚/零风险/包赚/guaranteed/躺赚"等禁用词**（已审计）。
- **建议**：当前 FAQ 偏采购选型，后续可补充招商政策类问答（代理条件/返点/区域保护），每条答案首段 2-3 句给结论（AI 通常截首段引用）。

### P3-2 STATIC_URL 与 Vite base 不一致（部署注意事项）

- **现象**：Django `STATIC_URL='static/'`，但 Vite 构建产物引用 `/assets/...`，两者不匹配。
- **影响**：SERVE_SPA 模式下 Django 不自动服务 `/assets/`；生产部署需 Nginx 将 `/assets/` 映射到 `frontend/dist/assets/`。
- **建议**：部署文档中固化 Nginx 配置：`location /assets/ { root /path/to/frontend/dist; }`，或调整 Vite `base` 与 Django `STATIC_URL` 对齐。
