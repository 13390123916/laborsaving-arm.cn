# 部署文档 - 生产环境上线指南

> 适用环境：**阿里云万网 ECS** ｜ 域名 **laborsaving-arm.cn** ｜ 备案 **管局审核中**

> 架构：Nginx（反向代理 + 静态资源） + Gunicorn（Django 应用） + Django `SERVE_SPA`（同域托管 Vue 构建产物）

## 三、部署流程

### 1. 后端部署（Django + Gunicorn + Nginx）

```bash
cd backend
pip install -r requirements.txt
pip install gunicorn
python manage.py migrate
python manage.py collectstatic --noinput
python manage.py shell < api/init_data.py
```

启动 Gunicorn（建议用 Supervisor 守护）：
```bash
gunicorn config.wsgi:application -b 127.0.0.1:8000 --workers 4 --daemon
```

### 1.1 后端环境变量（生产必配）

项目已支持以下环境变量，覆盖安全与生产配置（默认均为开发态，上线前务必设置）：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DJANGO_SECRET_KEY` | 内置占位串 | 生产请设为高强度随机串 |
| `DEBUG` | `True` | 生产设为 `False` |
| `ALLOWED_HOSTS` | `laborsaving-arm.cn,www.laborsaving-arm.cn,localhost,127.0.0.1` | 已默认收敛为真实域名白名单；可用此变量精确覆盖（逗号分隔） |
| `SERVE_SPA` | `False` | 同域托管前端时设为 `True`（Django 直接服务 `frontend/dist/`） |
| `SECURE_SSL_REDIRECT` | `False` | 生产设为 `True`（强制 HTTPS） |
| `SECURE_COOKIES` | `False` | 生产设为 `True`（Cookie 仅 HTTPS） |
| `HSTS_SECONDS` | `0` | 生产建议 `31536000`（开启 HSTS） |

> 媒体文件由 Django 在 `DEBUG` 或 `SERVE_SPA` 模式下自动服务；生产若由 Nginx 托管 `/media/`，需在 Nginx 配置指向 `backend/media/`。

### 2. 前端构建 + SSG 预渲染

```bash
cd frontend
pnpm install
pnpm run build:prerender    # 构建 + 预渲染所有路由（需 Django 后端在 :8000 + Chromium）
```

构建产物位于 `frontend/dist/`，含：所有路由预渲染 HTML（真实正文 + JSON-LD）、哈希 JS/CSS、`robots.txt` / `sitemap.xml` / `llms.txt`。

> 预渲染前置：Django 后端先启动在 `127.0.0.1:8000`；若后台数据（资讯/FAQ/凭证）更新，需重新构建以刷新快照。

### 3. Nginx 配置（同域部署，推荐）

```nginx
server {
    listen 80;
    server_name laborsaving-arm.cn www.laborsaving-arm.cn;

    root /path/to/frontend/dist;
    index index.html;

    location = /robots.txt { }
    location = /sitemap.xml { }
    location = /llms.txt { }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /admin/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    location /static/ { alias /path/to/backend/staticfiles/; }
    location /media/  { alias /path/to/backend/media/; }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

> 说明：`SERVE_SPA=True` 时 Django 按路由返回预渲染快照（`config/urls.py` 的 `spa_fallback`）；Nginx 的 `try_files` 兜底 SPA 入口。两者配合确保爬虫拿到正确 TDK 与正文。

## 四、HTTPS 配置（备案通过后）

- 阿里云：在控制台「SSL 证书」申请免费 DV 证书，下载 Nginx 版本并配置；
- 或 Let's Encrypt：`certbot --nginx -d laborsaving-arm.cn -d www.laborsaving-arm.cn`
- 配置后设 `SECURE_SSL_REDIRECT=True`、`SECURE_COOKIES=True`、`HSTS_SECONDS=31536000`。

---

## 五、ICP 备案与上线步骤（管局审核通过后）

1. 管局审核通过，取得 ICP 备案号（如 `辽ICP备XXXXXXXX号-X`）。
2. 登录 Admin（`/admin`）→「站点配置」→ 填入 `icp_beian`（备案号），页脚自动展示可点击链接（指向工信部 `beian.miit.gov.cn`）。
3. 阿里云控制台将 `laborsaving-arm.cn`、`www.laborsaving-arm.cn` 解析到 ECS 公网 IP。
4. 申请并部署 HTTPS 证书。
5. 配置环境变量 `DEBUG=False SERVE_SPA=True SECURE_SSL_REDIRECT=True` 后重启 Gunicorn + Nginx。
6. 各站长平台提交 `sitemap.xml`（详见 `docs/SEO配置说明.md`）。

> ⚠️ 备案审核中：**不要提前公开解析域名**，保持开发/内网访问，避免备案被驳回。
