# 部署文档 - 生产环境上线指南

## 一、服务器环境要求

- 操作系统：Linux（Ubuntu 20.04+ / CentOS 7+）
- Python：3.11+
- Node.js：18+
- Web服务器：Nginx
- 应用服务器：Gunicorn

---

## 二、后端部署（Django + Gunicorn + Nginx）

### 1. 安装依赖
```bash
cd backend
pip install -r requirements.txt
pip install gunicorn
python manage.py migrate
python manage.py collectstatic --noinput
python manage.py shell < api/init_data.py
```

### 2. 启动 Gunicorn
```bash
gunicorn config.wsgi:application -b 127.0.0.1:8000 --workers 4 --daemon
```

### 3. Nginx 配置
```nginx
server {
    listen 80;
    server_name laborsaving-arm.cn www.laborsaving-arm.cn;

    location /static/ {
        alias /path/to/backend/staticfiles/;
    }
    location /media/ {
        alias /path/to/backend/media/;
    }
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    location /llms.txt { proxy_pass http://127.0.0.1:8000; }
    location /robots.txt { proxy_pass http://127.0.0.1:8000; }
    location /sitemap.xml { proxy_pass http://127.0.0.1:8000; }
    location /schema/ { proxy_pass http://127.0.0.1:8000; }
}
```

## 三、前端部署（Vue3 静态文件）

### 1. 构建
```bash
cd frontend
npm install
npm run build
```

### 2. Nginx 配置（SPA）
```nginx
server {
    listen 80;
    server_name laborsaving-arm.cn;

    root /path/to/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

> 推荐前后端同域名，API 走 `/api/` 路径代理，避免跨域问题。
